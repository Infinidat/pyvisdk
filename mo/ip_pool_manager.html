3.57. IpPoolManager¶

Property of

ServiceContent

See also

Datacenter, IpPool

Since

vSphere API 4.0

class pyvisdk.mo.ip_pool_manager.IpPoolManager(core, name=None, ref=None, type='IpPoolManager')[source]¶

IP Pools are used to allocate IPv4 and IPv6 addresses to vApps.

CreateIpPool(dc, pool)[source]¶

Create a new IP pool.The name field must be defined, all other fields are optional. If unset, they will be given default values.The ID for the pool is generated by the server and should not be defined on the pool object passed to this method.

Parameters:
  • dc – The datacenter on which to create the pool.
  • pool – The IP pool to create on the server
Return type:

xsd:int
DestroyIpPool(dc, id, force)[source]¶

Destroys an IP pool on the given datacenter.Looks up the pool on the datacenter by ID and deletes it. If the pool is in use, the method throws InvalidState unless the force flag is true.

Parameters:
  • dc – The datacenter on which to find the pool
  • id – The unique ID of the pool
  • force – If true, the pool will be destroyed even if it is in use
QueryIpPools(dc)[source]¶

Return the list of IP pools for a datacenter.

Parameters:dc – The datacenter for which to look up the IP pools.
Return type:IpPool[]
UpdateIpPool(dc, pool)[source]¶

Update an IP pool on a datacenter.The pool to update is looked up from the value of the id field.All fields in the pool except the id are optional. Only defined values are stored on the server.

Parameters:
  • dc – The datacenter on which to look up the pool.
  • pool – The IP pool to update on the server
