3.5. ClusterProfile¶

See also

ClusterProfileConfigSpec

Since

vSphere API 4.0

Extends

Profile

class pyvisdk.mo.cluster_profile.ClusterProfile(core, name=None, ref=None, type='ClusterProfile')[source]¶

Properties

AssociateProfile(entity)¶

Associate a profile with a managed entity.

Parameters:entity – The entity(s) to associate with the profile. If entity is already associted with

the profile, association is maintained and operation is treated as a no- op. throws InvalidType If the entity is of an unexpeted type. throws InvalidArgument If the association conflicts with existing association.

CheckProfileCompliance_Task(entity)¶

Check compliance of an entity against a Profile.

Parameters:entity – If specified, the compliance check is done against this entity. If the entity is

not specified, a compliance check will be run on all the entities associated with the profile. Entity need not be associated with the profile.

Return type:Task
DestroyProfile()¶

Destory the Profile

DissociateProfile(entity)¶

Dissociate a profile from a managed entity.

Parameters:entity – Entity(s) from which to dissociate the profile. If unset, the profile is

dissociated from all managed entities it is currently associated with. If the specified entity is not associated with the profile, the operation is a no-op. throws InvalidArgument If the dissociation conflicts with existing association.

ExportProfile()¶

Export the profile into a serialized form. The serialized string can then be used to create a profile using SerializedCreateSpec

Return type:xsd:string
complianceStatus¶

Overall compliance of entities associated with this profile. If one of the entities is out of compliance, profile is out of compliance. If all entities are in compliance, profile is in compliance. If compliance status of one of the entities is unknown, compliance status of the profile is unknown. See
